Project 5: File I/O (Reading and Writing Files)
Real-Life Problem

You build a to-do list program. The user adds tasks like "Buy milk" and "Call doctor". The program runs, shows the tasks, and lets the user add new ones.

But when the program closes, all tasks disappear.

You need a way to save tasks between runs. Environment variables cannot do this (they die when the process ends). You need files.
What You Will Learn
Concept	What it is
File I/O	Input/Output from/to files
open() in Python	Open a file for reading/writing
Modes	'r' (read), 'w' (write), 'a' (append)
with statement	Automatically closes files
> >> in Bash	Redirect output to files
< in Bash	Read input from files
cat	Display file contents
Absolute vs relative paths	Where files live in the filesystem

Problem: File permissions in Linux are represented by three octal digits (0-7). Each digit is the sum of read(4), write(2), execute(1) permissions.

Examples:

    7 = 4+2+1 = rwx

    6 = 4+2 = rw-

    5 = 4+1 = r-x

    0 = ---

For a file with permission 754:

    What does the first digit (7) mean for the owner? (rwx? rw-? etc.)

    What does the second digit (5) mean for the group?

    What does the third digit (4) mean for others?

    Write the full permission string (like rwxr-xr--)

Bash Bootcamp Addition

In your mdu_bash_bootcamp folder, create a new script called file_redirection.sh that demonstrates:
bash

#!/usr/bin/env bash
# Demonstrates > (overwrite), >> (append), < (read input from file)

echo "Line 1" > test.txt          # overwrites or creates
echo "Line 2" >> test.txt         # appends
cat test.txt                      # displays content
while read line; do
    echo "Read: $line"
done < test.txt                   # redirects file content as input

Run it and understand each line.
